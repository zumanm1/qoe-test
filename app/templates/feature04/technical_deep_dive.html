Radio Access Network (RAN) Technologies

The Radio Access Network (RAN) is the part of a mobile telecommunications system that connects user equipment to the core network through radio connections.

LTE RAN Architecture
eNodeB Components
  • Radio Resource Control (RRC)
  • Packet Data Convergence Protocol (PDCP)
  • Radio Link Control (RLC)
  • Medium Access Control (MAC)
  • Physical Layer (PHY)
eNodeB Interfaces
  • LTE-Uu: Interface between UE and eNodeB
  • X2: Interface between eNodeBs
  • S1-MME: Control plane between eNodeB and MME
  • S1-U: User plane between eNodeB and S-GW
Radio Resource Management
Function Description Impact on QoE
Radio Bearer Control Management of radio bearers (establishment, maintenance, release) Direct impact on connection establishment delay and service availability
Radio Admission Control Decides whether new radio bearer requests are accepted Affects congestion and service quality during high load
Connection Mobility Control Manages mobility of radio connections (handovers) Critical for maintaining service continuity during movement
Dynamic Resource Allocation Scheduler function that allocates and de-allocates resources to UEs Directly impacts throughput, latency, and jitter
Inter-cell Interference Coordination Manages interference between cells to optimize performance Affects edge-user performance and overall capacity
Physical Layer Technologies
Downlink: OFDMA

Orthogonal Frequency Division Multiple Access (OFDMA) is used in LTE downlink transmission:

  • Divides bandwidth into multiple narrowband subcarriers (15 kHz spacing)
  • Enables multi-user scheduling in both time and frequency domains
  • Provides resistance to multipath fading
  • Supports flexible bandwidth allocations (1.4, 3, 5, 10, 15, 20 MHz)
Uplink: SC-FDMA

Single Carrier Frequency Division Multiple Access (SC-FDMA) is used in LTE uplink transmission:

  • Similar to OFDMA but with an additional DFT processing step
  • Provides lower Peak-to-Average Power Ratio (PAPR)
  • Improves power efficiency for the UE (extended battery life)
  • Requires contiguous resource block allocation for a single user
MIMO Technology in LTE

Multiple Input Multiple Output (MIMO) antenna systems increase spectral efficiency:

  • Transmit Diversity: Improves signal robustness through redundancy
  • Spatial Multiplexing: Increases data rates by transmitting different data streams in parallel
  • Beamforming: Focuses signal energy toward specific users to improve SNR
  • 2x2 MIMO: Basic configuration in LTE with 2 antennas each at transmitter and receiver
  • 4x4 MIMO: Advanced configuration supporting higher throughput
  • Massive MIMO: Uses large numbers of antennas (primarily in 5G) for improved capacity
IP-MPLS Transport Network Architecture

Multiprotocol Label Switching (MPLS) is a high-performance packet forwarding technology that integrates the intelligence of routing with the performance of switching to create an efficient transport mechanism for mobile backhaul and core networks.

Mobile Network Domain Structure

The mobile network comprises several interconnected domains, each with specific functions and equipment:

RAN Domain

Radio Access Network components:

  • UE (User Equipment): Mobile devices connecting to the network
  • eNodeB: Base stations providing radio connectivity to UEs

The RAN domain connects users to the mobile network and handles radio resource management.

EUTRAN Domain

Evolved Universal Terrestrial Radio Access Network components:

  • UE: User devices
  • eNodeB: LTE base stations
  • S-GW (Serving Gateway): Routes user data between eNodeB and P-GW
  • P-GW (Packet Data Network Gateway): Connects to external networks
  • IGW: Internet Gateway providing external connectivity
Internet Domain

Internet connectivity components:

  • PE1 (Provider Edge 1): Edge router connecting to international networks
  • P0 (Provider Core): Core router providing transit services
  • PE2 (Provider Edge 2): Edge router connecting to content providers
  • Ookla Server: Speed test server for network performance testing
Core Domain

Mobile Core Network components:

  • MME (Mobility Management Entity): Controls signaling for mobility and security
  • HSS (Home Subscriber Server): Central user database
  • PCRF (Policy and Charging Rules Function): Policy enforcement
  • S-GW/P-GW: User plane gateways
MPLS Network Components
  • Label Edge Router (LER/PE): Entry/exit points to the MPLS network
  • Label Switch Router (LSR/P): Core MPLS routers that forward packets based on labels
  • Label Distribution Protocol (LDP): Establishes label mappings for routes
  • Resource Reservation Protocol (RSVP-TE): Used for traffic engineering
  • Label Switched Path (LSP): Predetermined path through the MPLS network
Benefits of MPLS in Mobile Networks
  • Traffic Engineering: Enables precise control over traffic flows
  • QoS Support: Provides differentiated service levels for various traffic types
  • Fast Reroute: Sub-50ms protection switching for high availability
  • Scalability: Supports large-scale networks with millions of routes
  • Service Convergence: Single network for multiple services (voice, data, video)
IP-MPLS Domain Structure

The Transport network consists of three distinct IP-MPLS domains, each serving different segments of the mobile network:

IP-MPLS Backhaul

Function: Connects RAN side via IP VRF to the Core Data Network

  • Utilizes Microwave (10Mbps - 2.5Gbps) and Fiber (100Mbps - 100Gbps) transport mediums
  • Connects to Core Data Network via Inter-AS Option A
  • Router naming convention: PEx1--Px0, PExx2

Primary Role: Aggregate traffic from eNodeBs and transport to Core Data Network

IP-MPLS Core Data Network

Function: Provides connectivity between Core Network elements

  • Connects S-GW, P-GW, and IGW
  • Delivers L2VPN and L3VPN services for Core Network traffic
  • Router naming convention: PEyy1, Pyy0, PEyy2

Primary Role: High-capacity transport between Core Network elements with strict QoS guarantees

IP-MPLS International/Internet

Function: Provides connectivity to external networks and internet

  • Router naming convention: PE1, P0, PE2
  • Direct connection to Ookla speed test servers
  • Provides peering with international carriers and content providers

Primary Role: International connectivity and internet access for mobile subscribers

MPLS VPN Services for Mobile Networks
L3VPN (Layer 3 VPN)

Layer 3 MPLS VPNs (RFC 4364/2547bis) provide logical network separation for different services:

  • Separate routing domains using VRF (Virtual Routing and Forwarding)
  • Used for separating S1-U, S1-C, and X2 traffic
  • Supports overlapping IP addressing between different VPNs
  • Uses BGP for distributing VPN routes between PE routers
L2VPN (Layer 2 VPN)

Layer 2 MPLS services provide Ethernet or other L2 connectivity across MPLS network:

  • VPWS (Virtual Private Wire Service): Point-to-point L2 circuits
  • VPLS (Virtual Private LAN Service): Multipoint L2 connectivity
  • EVPN (Ethernet VPN): Enhanced L2 VPN with BGP control plane
  • Used for transparent backhaul of cell site traffic

Segment Routing MPLS

Segment Routing is a modern network architecture that simplifies MPLS operations:

  • Source Routing: Path selection at ingress node instead of hop-by-hop signaling
  • Simplified Control Plane: No need for LDP or RSVP-TE protocols
  • Traffic Engineering: Flexible path selection with Segment Routing Traffic Engineering (SR-TE)
  • Fast Reroute: TI-LFA (Topology Independent Loop-Free Alternate) for sub-50ms protection
  • Scalability: Reduced state in the network as state is maintained at the edge
Transport Mediums for Mobile Backhaul
Fiber Transport (100Mbps to 100Gbps)

Fiber optic transmission is the preferred medium for mobile backhaul, offering:

  • High Capacity: From 100Mbps up to 100Gbps per wavelength
  • Low Latency: Typically < 0.5ms per 100km
  • High Reliability: > 99.999% availability with proper protection
  • Technologies: GPON/EPON, Metro Ethernet, DWDM, OTN
  • Best For: High-density urban areas, data centers, and core networks
Deployment Types:
Dark Fiber Dedicated fiber pair with operator-owned equipment
Leased Line Managed service with SLA guarantees
WDM Multiple wavelengths over shared fiber infrastructure
Microwave Transport (10Mbps - 2.5Gbps)

Microwave transmission provides wireless backhaul options for locations where fiber is unavailable:

  • Medium Capacity: From 10Mbps to 2.5Gbps depending on frequency band
  • Medium Latency: Typically 0.1-1ms per hop
  • Weather Dependent: Rain fade impacts higher frequency bands
  • Frequency Bands: 6-42 GHz licensed, 60-80 GHz E-band
  • Best For: Rural areas, difficult terrain, temporary deployments
Key Microwave Technologies:
Traditional (6-42 GHz) 50-500 Mbps, 5-15km range
E-Band (70-80 GHz) 1-2.5 Gbps, 1-5km range
MIMO & CA Multiple input/output and carrier aggregation for capacity boost
